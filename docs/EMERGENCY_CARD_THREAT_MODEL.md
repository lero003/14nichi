# 緊急カード脅威モデルと最小データ方針

最終更新: 2026-07-20  
対象: Slice 4「個人情報と出力」  
関連: `docs/FULL_MVP_COMPLETION_BRIEF.md`, `docs/PROJECT_FOUNDATION.md`, `14日note.md` §9.10–9.11

この文書は、個人情報を保存するコードを書く前に確定する実行仕様です。実装はこの境界を超えません。

## 1. 目的と非目的

### 目的

- 緊急時に、家族・支援者・利用者が **30秒以内** で連絡先・集合場所・アレルギー等の最小情報を確認できる。
- 端末紛失・画面のぞき見・共有・印刷時に、不要な個人情報を広げない。
- 利用者の明示操作なしに、個人情報を生成・共有・印刷・外部送信しない。

### 非目的（MVPでは扱わない）

- iCloud / クラウド同期、アカウント、外部サーバー送信
- 解析・広告・常時位置情報
- ロック画面ウィジェットや Apple Watch への緊急カード表示
- QR コード共有、多人数同時編集、暗号化メール送信
- 健康保険証番号・マイナンバー・詳細な診療録の保管

## 2. 保護対象

| 区分 | 例 | 感度 |
|---|---|---|
| 連絡・集合 | 緊急連絡先の氏名・電話、集合場所、避難予定場所 | 高 |
| 健康関連 | アレルギー、常用薬（名称レベルの自由記述） | 最高 |
| 任意メモ | ブレーカー等の短い注意メモ | 中〜高 |
| 表示名 | カード上の呼び名・氏名（任意） | 中 |
| 生成物 | PDF、印刷プレビュー、共有シートに載る本文 | 出力内容に依存 |
| 非保護（本スライス外） | お気に入り記事ID、備蓄数量、同梱ガイド本文 | 別境界 |

備蓄データと緊急カードは **別ストア・別スキーマ・別移行計画** とする。お気に入りは記事IDのみで個人情報と混ぜない。

## 3. 想定攻撃者とシナリオ

| 想定 | 例 | 主対策 |
|---|---|---|
| 端末の物理取得 | 紛失・盗難後に端末ロックを突破、またはロック前に触る | OSの端末ロック + データ保護。任意のアプリ内再認証 |
| 肩越しのぞき見 | 電車内・避難所で画面が見える | 非アクティブ時の秘匿オーバーレイ。大きな文字でも不要項目を出さない |
| 共有・印刷の取り違え | 誤って連絡先入りPDFを他人へ渡す | 項目の明示選択、出力直前の同意文、既定は個人情報オフ |
| バックアップからの抽出 | 暗号化されていないバックアップ、共有Mac | 端末内のみ・同期なし。iCloud同期を実装しない |
| ログ・クラッシュ報告 | フィールド値をログに書く | 値をログ・エラー文・ファイル名に載せない |
| 一時ファイル残存 | PDFをCachesに残す | 専用一時領域、短寿命、後片付けテスト |
| 悪意ある他アプリ | サンドボックス外読み取り | OSサンドボックスに依存。独自ネットワーク送信を持たない |
| 認証不能 | Face ID失敗・パスコード忘れ | データは消さない。閲覧できない旨だけ表示。端末パスコードはOS側 |

## 4. 最小データ（採用 / 不採用）

企画書 §9.10 の候補を、**緊急利用30秒** と **漏えい時の被害** で絞り込む。

### 採用（MVP）

| 項目 | 理由 | 制約 |
|---|---|---|
| 表示名（任意） | 複数人で端末を共有する場合の識別 | 短文。必須にしない |
| 緊急連絡先（0〜5件） |  immediate に電話するため | 表示名 + 電話番号（文字列）。関係の短いラベル任意 |
| 集合場所（任意・短文） | 家族の合流に直結 | 住所の精密さは利用者判断。自由記述上限あり |
| 避難予定場所（任意・短文） | 同上 | 同上 |
| アレルギー（任意・短文） | 救急時に致命的になりうる | 診断書の代替ではない旨をUIで明示 |
| 常用薬（任意・短文） | 救急・避難時の持参確認 | 用量・処方番号は求めない。名称レベル |
| 注意メモ（任意・短文） | 元栓・ブレーカー等を1欄に集約 | 保険証・暗証番号を書かない注意を表示 |

件数・文字数上限は実装で定数化し、長大な生活史の保存を防ぐ。

### 不採用（MVP）と理由

| 項目 | 理由 |
|---|---|
| 生年月日 | 30秒行動に不要。なりすまし・識別子リスク |
| 自宅住所の専用欄 | 集合・避難の自由記述で足りる。常時保存する固定住所は漏えい被害が大きい |
| 家族構成の人数詳細 | 備蓄側で扱う。カードには不要 |
| 持病の詳細診断名専用欄 | アレルギー・薬・注意メモに必要最小を書く。病歴DBにしない |
| 血液型 | 現代の救急現場では優先度が低く、誤記リスクがあるため既定項目にしない |
| かかりつけ医・健康保険情報 | 感度が高く、初期行動に必須でない |
| ペット・車両情報 | 初期MVPの30秒条件外。注意メモで足りる場合のみ |
| 水道・ガス・ブレーカーの個別必須欄 | 注意メモ1欄に集約 |
| 備蓄保管場所 | 備蓄機能側。PDF出力時に備蓄を選んだ場合のみ載せる |
| 顔写真・身分証画像 | 対象外 |

## 5. 保存境界

- **ストア**: 緊急カード専用 SwiftData ストア（ファイル名・スキーマ・MigrationPlan を備蓄と分離）。
- **場所**: アプリの Application Support 配下。iCloud / CloudKit 設定は付けない。
- **移行**: `EmergencyCardSchemaV1` と空の migration stages から開始。破壊的変更時は明示的な migration stage を追加する。
- **保存失敗**: `save()` は明示し、失敗を握りつぶさない。UIはエラーを表示する。
- **削除**: 「緊急カードをすべて削除」でカード本体・連絡先を消去し、コンテナ再生成後も復活しないことをテストする。備蓄・お気に入りは巻き込まない（全体リセットが必要なら別操作として明示）。

## 6. 認証・ロック・可用性

緊急時の可用性と保護は衝突するため、次を既定とする。

| 状況 | 既定動作 |
|---|---|
| アプリ内ロック | **既定オフ**（端末のロックとOSデータ保護を主防御とする） |
| 利用者がオンにした場合 | カード本文の表示・編集・PDFに個人情報を含める直前に、端末の生体認証またはデバイスパスコードを要求 |
| 認証キャンセル / 失敗 | データを消さない。本文は秘匿したまま。再試行を促す |
| 認証不能（未設定・非対応） | オプションをオンにできない、またはOSが提供するデバイス認証にフォールバック。迂回する秘密の質問は持たない |
| バックグラウンド / 非アクティブ | 個人情報画面をぼかしまたはプレースホルダで覆う。App Switcher への平文露出を抑える |
| フォアグラウンド復帰 | アプリ内ロックがオンなら再認証まで本文を出さない。オフなら秘匿を解除 |
| 端末ロック中 | アプリは動作しない前提。ロック画面へのカード露出は実装しない |

Face ID / Touch ID を「暗号化キーの唯一の解除条件」にはしない。OSの LocalAuthentication は **閲覧ゲート** として使い、保存データ自体はアプリサンドボックス内の通常保護に置く。紛失時の主防御は端末パスコードと「データ消去」設定である。

## 7. 画面・ログ・一時ファイル

- フィールド値を `print`、OSログ、解析、エラーアラートの詳細に載せない。
- 保存ファイル名・PDF既定ファイル名に氏名や電話を入れない（例: `14nichi-export.pdf`）。
- PDFは専用一時ディレクトリに書き、共有完了・キャンセル・失敗後に削除する。削除失敗は握りつぶさず、可能なら再試行する。
- スクリーンショット防止の完全保証はOSではできない。UI上の注意と秘匿オーバーレイで抑止する。

## 8. PDF・印刷・共有

- 出力対象は利用者がチェックした項目だけ。
- **既定**: 個人情報カテゴリ（連絡先・集合/避難・アレルギー・薬・注意メモ・表示名）はオフ。備蓄チェックリストは利用者が選んだ場合のみ。
- 生成ボタンの直前に、「紙やファイルに個人情報が残ること」「共有先を利用者が選ぶこと」を説明し、明示タップなしでは生成しない。
- ガイド記事本文の転載PDFは、著作権と未監修リスクがあるためMVPでは入れない（行動ガイドはアプリ内オフライン閲覧を正とする）。
- オフラインで生成できること。空選択は生成せず案内する。

## 9. バックアップ・同期・ネットワーク

- 緊急カードの iCloud 同期は実装しない（企画の「任意」は採用しない）。
- アカウント、外部API、広告、解析SDKを追加しない。
- 公式リンクを開く操作だけが通信境界。リンク集のURLは同梱の機械可読データで、Web本文は転載しない。

## 10. アクセシビリティと緊急UX

- VoiceOver で連絡先の電話番号を読み上げ可能にする（利用者が表示を許可した状態で）。
- Dynamic Type・ダークモード・Reduce Motion・キーボード操作を壊さない。
- 「大きい文字で少ない項目」を優先し、一覧→詳細の階層を浅くする。
- 未入力は空でよく、必須項目でブロックしない（緊急時に未完成カードでも使える）。

## 11. 実装チェックリスト（受け入れ）

- [ ] 採用項目以外を保存モデルに持たない
- [ ] 備蓄・お気に入りとストアが分離している
- [ ] 作成・閲覧・編集・削除・全削除ができる
- [ ] 保存失敗がUIに出る
- [ ] アプリ内ロック任意、既定オフ
- [ ] 非アクティブ時に秘匿する
- [ ] PDFは項目選択 + 同意後のみ。除外項目が混入しない
- [ ] 一時ファイル後片付けのテストがある
- [ ] 実ファイルストアで保存→再オープン→削除のテストがある
- [ ] 値をログ・ファイル名に出さない
