# フルMVP 手動スモーク手順

最終更新: 2026-07-20  
対象ビルド: マーケティングバージョン `0.1.0` / ビルド `1`

自動テストを通したあとに、実機またはシミュレータで実施する。結果は実施日・端末・所要時間を記録する。

## 前提

- 同梱記事はすべて `draft`（制作フィクスチャ）。正式な安全情報として扱わない。
- 機内モード相当でコールド起動できること（ガイド本文はオフライン）。
- 公式リンクはオンラインが必要。

## A. コールド起動と30秒到達（必須・実機）

1. アプリを完全終了する。
2. 機内モードをオンにする（または通信を切る）。
3. 起動し、ホーム相当（ガイドタブ）が表示されるまで測る。
4. 状況「地震が起きた」→ 記事を開き、最初の行動見出しまで到達する。
5. **合格**: 起動開始から最初の行動確認まで **30秒以内**。
6. 記録欄: 端末 / OS / 所要秒 / 実施者 / 日付

## B. ガイド・探索

1. 状況切替で記事一覧が変わる。
2. 全文検索で本文ヒットする（例: 記事内の語）。
3. カテゴリ・行動時期フィルタをかけ、解除できる。
4. お気に入り登録 → 絞り込み → 再起動後も残る。
5. 未監修バナーが `draft` 記事に表示される。
6. 情報源の公式リンクは機内モードでは開けず、オンラインではブラウザへ進む。

## C. 備蓄・買い物

1. 人数と計画日数を変更して保存される。
2. 1人1日量と現在量を入れ、必要量・不足が表示される。
3. 期限を設定し、期限切れ/近い表示を確認する。
4. 買い物タブで不足のみ表示される。
5. 「在庫へ反映」後、不足が減り準備済みになる。
6. アプリ再起動後も値が残る。

## D. 緊急カード（個人情報）

1. 編集で連絡先・集合場所・アレルギー等を保存できる。
2. 文字数上限を超える入力が切り詰められる（または入力しにくい）。
3. 表示前認証をオンにすると、再表示に端末認証が必要。
4. 認証キャンセルで本文が漏れない。
5. ホームへ送って App Switcher で個人情報が平文で読めない（秘匿オーバーレイ）。
6. 「すべて削除」後、再起動しても内容が戻らない。
7. 備蓄・お気に入りが消えていない。

## E. 公式リンク集

1. 「その他」→「公式情報リンク集」。
2. オンライン接続が必要である旨が表示される。
3. 内閣府・気象庁など https の公的リンクが一覧される。
4. タップで外部ブラウザ/アプリが開く（通信境界はこの操作のみ）。

## F. PDF・印刷

1. 「その他」→「PDF・印刷」。
2. 既定では個人情報トグルがオフ。
3. 備蓄チェックリストのみ選択して生成できる。
4. 個人情報をオンにすると確認トグルなしでは生成できない。
5. 生成物のファイル名が `14nichi-export.pdf` 系で氏名を含まない。
6. 未選択項目の内容がPDFに出ない。
7. iPhone: 共有シート。Mac: 印刷/プレビュー導線。
8. 共有キャンセル後もアプリが落ちない。一時ファイルが残らない（可能なら確認）。

## H. 失敗系

1. 空の緊急カードでもクラッシュしない。
2. PDFで項目未選択時は生成されない。
3. 備蓄保存領域が壊れている想定は自動テスト側。手動では保存エラー表示があれば記録。
